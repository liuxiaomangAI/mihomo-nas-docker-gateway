# 🚀 Mihomo NAS 全流程终极部署手册（最终完整版）

更新时间：2026-03-30

---

# 🧠 一、系统目标

构建一套：

- 全网透明代理（TUN）
- DNS 无污染
- 自动更新节点
- 自动更新 GEO
- 可恢复 / 可迁移 / 可长期运行

---

# 🏗️ 二、最终架构

主路由 → NAS（Mihomo）→ 全部设备

NAS IP：192.168.5.21（或者你的NAS的在局域网的实际IP）

---

# ❗三、核心原则（必须理解）

1. Geo 文件解决“启动死锁”
2. proxy-server-nameserver 解决“节点解析死锁”
3. proxy-providers 解决“节点自动更新”
4. geo-auto-update 解决“长期维护”

---

# 📦 四、准备 4 个 Geo 文件（必须）

在 Windows 下载：

GeoSite：
https://github.com/MetaCubeX/meta-rules-dat/releases/download/latest/geosite.dat

GeoIP：
https://github.com/MetaCubeX/meta-rules-dat/releases/download/latest/geoip-lite.dat

country-lite.mmdb：
https://github.com/MetaCubeX/meta-rules-dat/releases/download/latest/country-lite.mmdb

ASN：
https://github.com/MetaCubeX/meta-rules-dat/releases/download/latest/GeoLite2-ASN.mmdb

---

⚠️ 关键说明（必须先做）

这 4 个 Geo 文件必须在 Mihomo 第一次启动前手动放入 NAS。

否则会出现：
- 启动卡死
- Geo 下载失败
- context deadline exceeded

原因：
NAS 初始无法翻墙，Mihomo 无法从 GitHub 下载 Geo 文件。

# 📦 五、准备 1 个翻墙订阅链接（必须）

需要准备一个 Clash / Mihomo 兼容的订阅链接。

说明：
- 支持 SSR / V2Ray / Trojan / Hysteria 等协议
- 必须包含可用节点

我推荐一个翻墙网站（我一直稳定在用，个人推荐节点：[旗舰版+]日本4）：https://www.xcjs123.com/auth/register?code=EsTMyvDQhJMjC5Lp。从此网站订阅后，就能获得一个订阅链接。

# ⚙️ 六、config.yaml（最终版）

```yaml

mixed-port: 7890
allow-lan: true
bind-address: "*"
mode: rule
log-level: info
ipv6: false
tcp-concurrent: true
unified-delay: true

external-controller: 0.0.0.0:9090

profile:
  store-selected: true
  store-fake-ip: true

geodata-mode: true

# 🔥 自动更新 Geo（加在这里）
geo-auto-update: true
geo-update-interval: 24

# 🔥 Geo 数据源（必须和自动更新一起）
geox-url:
  geoip: "https://github.com/MetaCubeX/meta-rules-dat/releases/download/latest/geoip-lite.dat"
  geosite: "https://github.com/MetaCubeX/meta-rules-dat/releases/download/latest/geosite.dat"
  mmdb: "https://github.com/MetaCubeX/meta-rules-dat/releases/download/latest/country-lite.mmdb"
  asn: "https://github.com/MetaCubeX/meta-rules-dat/releases/download/latest/GeoLite2-ASN.mmdb"

sniffer:
  enable: true
  sniff:
    HTTP:
      ports: [80, 8080-8880]
      override-destination: true
    TLS:
      ports: [443, 8443]
    QUIC:
      ports: [443, 8443]
  skip-domain:
    - "Mijia Cloud"
    - "+.push.apple.com"

tun:
  enable: true
  stack: mixed
  dns-hijack:
    - "any:53"
    - "tcp://any:53"
  auto-route: true
  auto-redirect: true
  auto-detect-interface: true
  strict-route: true

dns:
  enable: true
  listen: 0.0.0.0:1053
  ipv6: false
  use-hosts: true
  use-system-hosts: true
  enhanced-mode: fake-ip
  fake-ip-range: 198.18.0.1/16
  fake-ip-filter:
    - "+.lan"
    - "+.local"
    - "localhost"
    - "+.home.arpa"
    - "localhost.ptlogin2.qq.com"

  default-nameserver:
    - tls://223.5.5.5
    - tls://223.6.6.6

  nameserver:
    - https://doh.pub/dns-query
    - https://dns.alidns.com/dns-query

  fallback:
    - https://1.1.1.1/dns-query
    - https://8.8.8.8/dns-query

  fallback-filter:
    geoip: true
    geoip-code: CN

  proxy-server-nameserver:
    - https://doh.pub/dns-query
    - https://dns.alidns.com/dns-query

proxy-providers:
  lazydoge:
    type: http
    url: "实际的订阅链接替换此处"
    interval: 86400
    path: ./providers/lazydoge.yaml
    health-check:
      enable: true
      url: "https://www.gstatic.com/generate_204"
      interval: 300


proxy-groups:
  - name: 全部节点
    type: select
    include-all: true
    exclude-type: direct

  - name: 日本节点
    type: select
    include-all: true
    exclude-type: direct
    filter: "(?i)日|jp|japan|tokyo|osaka|东京|大阪"

  - name: OpenAI
    type: select
    proxies:
      - 日本节点

  - name: Telegram
    type: select
    proxies:
      - 日本节点

  - name: 默认代理
    type: select
    proxies:
      - 日本节点
      - 全部节点
      - DIRECT

  - name: 国内网站
    type: select
    proxies:
      - DIRECT
      - 默认代理

rules:
  - GEOIP,LAN,DIRECT,no-resolve

  - GEOSITE,CN,国内网站
  - GEOIP,CN,国内网站

  - DOMAIN-SUFFIX,openai.com,OpenAI
  - DOMAIN-SUFFIX,api.openai.com,OpenAI
  - DOMAIN-SUFFIX,chatgpt.com,OpenAI
  - DOMAIN-SUFFIX,oaistatic.com,OpenAI
  - DOMAIN-SUFFIX,oaiusercontent.com,OpenAI
  - DOMAIN-KEYWORD,openai,OpenAI

  - GEOSITE,telegram,Telegram
  - GEOIP,telegram,Telegram

  - MATCH,默认代理
```

⚠️ 核心说明（必须保留）

proxy-server-nameserver 用于专门解析代理节点域名。

如果没有这一项，可能出现：

- 节点域名解析失败
- fallback 走 8.8.8.8 超时
- 代理无法建立

这是本方案能成功运行的关键配置之一。

services:

    container_name: mihomo
    restart: unless-stopped

    privileged: true

      - NET_ADMIN
    volumes:
      - /vol1/1000/Docker/mihomo:/root/.config/mihomo

## 🐳 在飞牛 NAS 中部署 Mihomo（Docker Compose）

1. 打开飞牛 NAS 的 **Docker 管理界面**

2. 进入：

   ```
   Compose → 新增项目
   ```

3. 填写项目信息：

   * **项目名称**：`mihomo`（或任意自定义名称）
   * **项目路径**：选择

     ```
     /vol1/1000/Docker/mihomo
     ```

4. 系统会自动弹出提示：

   > 所选择的路径已包含 docker-compose 配置文件
   > 是否使用现有配置文件创建项目？

   👉 点击：**确认**

5. 在项目配置界面：

   * 检查 `docker-compose.yml` 是否正确加载
   * 无需修改内容

6. 点击右下角：

   ```
   确认 → 部署
   ```
